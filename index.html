How This Report Is Generated

1
Scrape ESMA page — The eu-crypto-register-monitor workflow fetches the MiCA register page and extracts CSV download URLs via regex (URLs shift when ESMA updates files).
2
Download CSVs — Fetches 4 CSV files (CASPs, EMT, ART, Non-Compliant) through the corporate proxy. White papers omitted by default.
3
Parse & normalise — Entity keys derived from name + LEI. Country codes mapped to full names. Service codes parsed into human-readable labels.
4
Diff against snapshot — Previous run's entity-key set compared to current to detect additions, withdrawals, and country-level deltas.
5
Output — Structured JSON with summary stats, changes, and next_run_snapshot blob. This HTML is generated from that output.

Integration: The gtm-regulatory-outlook workflow pulls MiCA register data alongside media monitor and competitor intel feeds when generating the EMEA quarterly edition.

Data freshness: ESMA updates CSVs ~weekly. Last ESMA update: 4 June 2026.
